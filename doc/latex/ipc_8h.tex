\doxysubsection{Detailed Description}
Defines the common interface and base structure for IPC mechanisms. 

author Animesh0817 (\href{mailto:mailtome.anni@gmail.com}{\texttt{ mailtome.\+anni@gmail.\+com}})

This file provides the interface that all IPC mechanisms should implement. It defines the {\ttfamily \mbox{\hyperlink{structipc__handle__t}{ipc\+\_\+handle\+\_\+t}}} structure, which serves as a common base for different IPC mechanisms, and a set of function pointers to be implemented by each IPC type.

{\footnotesize\ttfamily struct \mbox{\hyperlink{structipc__handle__t}{ipc\+\_\+handle\+\_\+t}}}



Structure representing an IPC handle. 

The \mbox{\hyperlink{structipc__handle__t}{ipc\+\_\+handle\+\_\+t}} structure serves as a generic handle for IPC mechanisms. It contains function pointers that define how an IPC handle is initialized, how data is sent and received, and how the IPC handle is destroyed. This abstraction allows different IPC implementations (e.\+g., sockets, shared memory, etc.) to use the same interface, making the framework flexible and extendable.

Members\+:
\begin{DoxyItemize}
\item init\+: Function pointer for initializing the IPC handle.
\item send\+: Function pointer for sending data using the IPC handle.
\item receive\+: Function pointer for receiving data using the IPC handle.
\item destroy\+: Function pointer for destroying the IPC handle and releasing associated resources.
\end{DoxyItemize}

Create a new IPC handle. 

This function allocates memory for a new Inter-\/\+Process Communication (IPC) handle and returns a pointer to it. The IPC handle is an abstract representation used to manage different types of IPC mechanisms (e.\+g., sockets, shared memory, etc.).

This function does not initialize the handle beyond basic memory allocation. The user is responsible for assigning or initializing the relevant function pointers (e.\+g., init, send, receive, destroy) after creation.

\begin{DoxyReturn}{Returns}
Pointer to the created IPC handle. Returns {\ttfamily NULL} if memory allocation fails.
\end{DoxyReturn}
\begin{DoxyNote}{Note}
The returned handle must be freed using {\ttfamily ipc\+\_\+destroy} to avoid memory leaks.
\end{DoxyNote}

Destroy an IPC handle and release allocated memory. 

This function frees the memory associated with a given IPC handle. It is the responsibility of the caller to ensure that the handle is no longer in use when this function is called, as calling this function will deallocate the memory and invalidate the pointer.
